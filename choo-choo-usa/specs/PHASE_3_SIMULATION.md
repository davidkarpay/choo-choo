# Phase 3: The Simulation Layer

**Build after Phase 2.** This phase adds the economic heartbeat -- cargo flowing, passengers traveling, industries producing and consuming, and a database tracking it all.

---

## Overview

Phase 3 transforms Choo-Choo USA from an animated map into a living simulation. Cargo is generated by industries, loaded onto trains, transported across corridors, and delivered to destinations. Passengers board at stations, ride to their destinations, and disembark. The player can explore all of this data through panels and dashboards.

This is the "child's model of the entire industry" -- simplified but real. A kid should be able to understand that coal comes from mines, goes on trains, and powers cities. That oranges grow in California and ride trains to tables in New York.

---

## Industry Model

Industries are the source and sink of cargo. Each industry produces or consumes specific cargo types.

### Industry Types

| Industry | Produces | Consumes | Regions |
|----------|----------|----------|---------|
| Coal Mining | Coal | (nothing) | Appalachia (West Virginia, Kentucky, Pennsylvania), Wyoming |
| Power Generation | (nothing) | Coal, fuel | Midwest, everywhere |
| Agriculture (Grain) | Wheat, corn, soybeans | (nothing) | Great Plains (Kansas, Nebraska, Iowa, Illinois) |
| Agriculture (Produce) | Fruits, vegetables | (nothing) | California, Florida, Texas |
| Livestock Ranching | Cattle, livestock | (nothing) | Texas, Montana, Wyoming, Nebraska |
| Automobile Manufacturing | Automobiles | Steel, parts | Detroit (Michigan), Tennessee, Alabama |
| Steel Production | Steel | Iron ore, coal | Indiana, Ohio, Pennsylvania |
| Oil Refining | Fuel, chemicals | Crude oil | Texas, Louisiana, California |
| Lumber / Forestry | Lumber | (nothing) | Pacific Northwest (Oregon, Washington), Southeast |
| Food Processing | Packaged food | Produce, grain, livestock | Everywhere (major cities) |
| Retail / Distribution | (nothing) | Everything | Major cities |
| Passenger Travel | Passengers (outbound) | Passengers (inbound) | All stations |

### Cargo Generation

Every simulation hour, the engine rolls for new cargo demands:

```
For each industry:
  if random() < industry.production_rate:
    create new CargoShipment {
      type: industry.produces,
      quantity: random(industry.min_batch, industry.max_batch),
      origin: industry.station,
      destination: random consumer station for this cargo type,
      priority: normal | express,
      created_at: simulation_time
    }
```

Cargo shipments wait at their origin station until a train with matching cargo capability arrives and has capacity.

### Cargo Loading and Delivery
1. Train arrives at station
2. Unload any cargo destined for this station (delivery complete, remove from manifest)
3. Load waiting cargo that matches this train's type and has capacity
4. Train departs

## Passenger Model

Passengers are individual named characters with origins, destinations, and visible behaviors.

### Passenger Generation

Passengers spawn at stations with randomized attributes:

```
Passenger {
  name: random from name database,
  age_group: child | adult | elderly,
  appearance: { hat, clothing_color, bag_type },  // for visual rendering
  origin_station: this station,
  destination_station: random reachable station,
  activity: waiting | boarding | sleeping | eating | reading | talking | looking_out_window,
  mood: happy | tired | excited | nervous
}
```

### Passenger Behaviors
Passengers cycle through activities during their journey:
- **Waiting** at origin station (on the platform, checking watch, sitting on bench)
- **Boarding** (walking from platform into the train car)
- **Riding** (one of: sleeping, eating, reading, talking to neighbor, looking out window)
- **Arriving** (gathering belongings, standing, walking to door)
- **Deboarding** (stepping off train onto destination platform)
- **Leaving** (walking away from the platform)

Activity transitions are random but weighted by time of day:
- Morning: more eating, reading, looking out window
- Afternoon: more talking, reading
- Evening: more sleeping, looking out window
- Night: mostly sleeping

## Simulation Dashboard

A toggleable overlay panel that shows the player the big picture:

### Cargo Overview
- Total tons moved today (by type)
- Top corridors by volume
- Pending shipments waiting for trains
- Delivery success rate
- Simple bar chart or pictograph showing cargo by type (use PixiJS or a React charting library styled to match the storybook)

### Passenger Overview
- Total passengers in transit
- Total passengers delivered today
- Busiest stations (by boarding/alighting)
- A scrollable list of passengers currently riding, with their names, origins, and destinations

### Industry Overview
- Map overlay showing industry icons at their locations
- Production/consumption rates
- Supply chain connections (arrows showing cargo flow from producers to consumers)

### Train Fleet Overview
- All 6 trains with their current status
- Cargo capacity vs. current load
- Lifetime statistics (total miles, total deliveries, total passengers carried)

## Data Persistence

All simulation state is saved to IndexedDB via Dexie.js:

### Tables
- `trains` -- current state of each train (position, cargo, status)
- `cargo_shipments` -- all active and completed shipments
- `passengers` -- all active and completed passenger journeys
- `statistics` -- aggregate counters (daily, all-time)
- `simulation_state` -- clock time, speed setting, player preferences

### Save/Load
- Auto-save every 30 seconds
- Manual save via button
- On load: check for saved state, offer "Continue" or "New Game"
- "New Game" clears IndexedDB and reloads seed data

## Event System

The simulation generates events that trigger narration, sound effects, and visual feedback:

| Event | Trigger | Result |
|-------|---------|--------|
| `train_departed` | Train leaves a station | Whistle sound, narrator text, map update |
| `train_arrived` | Train reaches a station | Bell sound, narrator text, cargo/passenger transfer |
| `cargo_loaded` | Cargo placed on train | Loading sound effect, manifest update |
| `cargo_delivered` | Cargo reaches destination | Celebration chime, statistics update |
| `passenger_boarded` | Passenger steps onto train | Door sound, passenger count update |
| `passenger_arrived` | Passenger reaches destination | Happy chime, "Welcome to [city]!" |
| `new_day` | Simulation clock hits midnight | Statistics reset for daily counters, roundhouse return sequence |
| `train_breakdown` | (rare random event) | Train stops, maintenance crew dispatched, narrator explains |

Events are emitted via a simple pub/sub (publish/subscribe) system and consumed by the UI, sound manager, and narrator.

## Technical Notes

### Simulation Loop Integration
Phase 3 adds cargo and passenger logic to the existing simulation tick loop from Phase 1/2:

```
Every tick:
  1. Advance clock
  2. Update train positions (Phase 2)
  3. Check for station arrivals/departures
  4. Process cargo loading/unloading (Phase 3 NEW)
  5. Process passenger boarding/deboarding (Phase 3 NEW)
  6. Generate new cargo demands (Phase 3 NEW)
  7. Generate new passengers (Phase 3 NEW)
  8. Emit events
  9. Update stores
```

### Performance Considerations
- Cap active cargo shipments at 500
- Cap active passengers at 300
- Archive completed shipments/journeys to a summary table, do not keep full records in memory
- Lazy-load passenger detail data only when viewing a specific train interior or station
